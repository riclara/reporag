# Reporag MCP Guidance

Use the `reporag` MCP conservatively for structured repository navigation before falling back to normal file exploration.

## Tool choice

- Use `get_symbol` for exact symbol lookup and symbol disambiguation.
- Use `find_callers` to answer who calls a symbol.
- Use `find_callees` to answer what a symbol calls.
- Use `search_code` for textual or natural-language search when no stable symbol is known yet.

## Disambiguation

- Prefer `path#symbol` when the same symbol exists in multiple files, for example `packages/app/src/mcp-server.ts#runMcpServer`.
- Use `Parent.symbol` for methods or nested members, for example `PlayerController.start`.
- Combine both as `path#Parent.symbol` when you need file and container disambiguation at the same time.

## Reading results

- `search_code` may report `route=symbol`, `route=fts`, or `route=hybrid`.
- `find_callers` and `find_callees` may include multiple `matchedSymbols` before the final `hits`; treat that as ambiguity to resolve first.
- Distinguish between an ambiguous selector and a true no-results case.

## Fallback

- If the MCP is unavailable, say so explicitly and continue with normal repo exploration.
- If you only see a `repository_rag` MCP that talks about PostgreSQL, snapshots, or `snapshot:index`, that is not the local reporag server generated by `init`; prefer `reporag`.
- If the results look stale or empty, suggest `reporag index`.
- If the MCP still does not help, inspect files directly.
